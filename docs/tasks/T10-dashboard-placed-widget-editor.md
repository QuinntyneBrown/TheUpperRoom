# T10 — Placed-widget editor state

**Phase**: 2 — Dashboard editor interaction model
**Area**: Dashboard
**Requirements**: L1-007, L2-033
**Source**: Screen-Level Missing Inventory — "Dashboard placed-widget editor state"

## Goal

Show the dashboard with widgets placed and the editor "edit mode" engaged: chrome around each widget that exposes drag, resize, and remove affordances.

## Scope

- Edit-mode toggle button (visible in the populated dashboard).
- Hover/active outline on each placed widget.
- Editor toolbar per widget: drag handle, resize handle, remove icon, optional settings cog.
- Done/Save and Cancel/Discard CTA in the page header.

## Acceptance criteria

- [ ] `Desktop / Dashboard - Edit Mode` exists alongside the populated dashboard.
- [ ] Each widget shows the editor chrome.
- [ ] Save and Cancel actions are visible.
- [ ] AA contrast on edit-mode chrome verified.
